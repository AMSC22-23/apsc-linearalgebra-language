Detailed Description

template<class Matrix, class Vector, ORDERINGTYPE ORDER_TYPE>
class apsc::LinearAlgebra::MPISparseMatrix< Matrix, Vector, ORDER_TYPE >

A class for parallel sparse matrix product.

Template Parameters
MatrixA sparse matrix class
VectorThe vector type used for the local solution and internal usages. It must have the following methods:
  • data(): Returns a pointer to the data buffer
  • resize(std::size_t): Resizes the vector with the requested length

◆ setup()

template<class Matrix , class Vector , ORDERINGTYPE ORDER_TYPE>
void apsc::LinearAlgebra::MPISparseMatrix< Matrix, Vector, ORDER_TYPE >::setup ( Matrix const &  compressed_global_sparse_matrix,
MPI_Comm  communicator 
)
inline

All processes will call setup but only the managing process (with number 0) will actually pass a non-empty global matrix. Currently only Eigen::SparseMatrix class is supported.

Template Parameters
ORDERThe input sparse matrix ordering type
Parameters
compressed_global_sparse_matrixThe global matrix in compressed mode (https://en.wikipedia.org/wiki/Sparse_matrix#Compressed_sparse_row_(CSR,_CRS_or_Yale_format))
communicatorThe MPI communicator
Note
This is not the only way to create a parallel matrix. Alternatively, all processes may directly build the local matrix, for instance reading from a file. In this case, for the setup we need just the number of rows and columns of the global matrix
